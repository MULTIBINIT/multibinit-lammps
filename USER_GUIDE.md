## Overview

This plugin allows LAMMPS to use effective potentials generated by Multibinit (part of the ABINIT package) for molecular dynamics simulations. The effective potential includes:

- **Harmonic terms**: Second-order force constants from DDB files
- **Anharmonic terms**: Higher-order polynomial coefficients from XML files
- **Dipole-dipole interactions**: Long-range electrostatic interactions

### Key Features

- Seamless integration with LAMMPS MD engines
- Support for NVT, NVE, and NPT ensembles
- MPI parallelization
- Automatic atom reordering between LAMMPS and Multibinit conventions

### File Requirements

| File | Purpose | Required |
|------|---------|----------|
| `*.DDB` | Harmonic force constants | Yes |
| `*.xml` | Anharmonic coefficients | Optional |

## Installation

### Prerequisites

- ABINIT (with shared library support)
- LAMMPS (with plugin support)
- CMake ≥ 3.10
- C++11 compatible compiler
- MPI (for parallel runs)

### Building ABINIT

```bash
git clone https://github.com/abinit/abinit.git
cd abinit
mkdir build_so && cd build_so

cmake .. \
  -DCMAKE_INSTALL_PREFIX=/path/to/install \
  -DBUILD_SHARED_LIBS=ON \
  -DENABLE_MULTIBINIT=ON

make -j4
make install
```

### Building the Plugin

```bash
cd mblammps
mkdir build && cd build

cmake .. \
  -DABINIT_ROOT=/path/to/abinit/install \
  -DLAMMPS_ROOT=/path/to/lammps

make -j4
```

The plugin library `multibinitplugin.so` will be created in the build directory.

### Environment Setup

Add to your `.bashrc` or run before using:

```bash
export DYLD_LIBRARY_PATH=/path/to/abinit/install/lib:$DYLD_LIBRARY_PATH
# Linux: export LD_LIBRARY_PATH=/path/to/abinit/install/lib:$LD_LIBRARY_PATH
```

## Quick Start

### Minimal Example

Create `in.minimal`:

```lammps
# Minimal Multibinit-LAMMPS input
units metal
dimension 3
boundary p p p
atom_style atomic

# Create 2x2x2 supercell of BaHfO3
lattice custom 4.1490 &
  a1 1.0 0.0 0.0 &
  a2 0.0 1.0 0.0 &
  a3 0.0 0.0 1.0 &
  basis 0.0 0.0 0.0 &
  basis 0.5 0.5 0.5 &
  basis 0.5 0.5 0.0 &
  basis 0.0 0.5 0.5 &
  basis 0.5 0.0 0.5

region supercell block 0 2 0 2 0 2
create_box 3 supercell
create_atoms 1 box &
  basis 1 1 &
  basis 2 2 &
  basis 3 3 &
  basis 4 3 &
  basis 5 3

mass 1 137.33  # Ba
mass 2 178.49  # Hf
mass 3 16.00   # O

# Load plugin
plugin load multibinitplugin.so

# Define potential
pair_style multibinit 2 2 2 ngqpt 2 2 2 dipdip 0
pair_coeff * * BaHfO3.DDB

# Run MD
fix 1 all nvt temp 300 300 0.1
timestep 0.001
run 1000
```

Run:
```bash
lmp -in in.minimal
```

## Input Structure

### LAMMPS Structure Setup

The plugin automatically maps LAMMPS atoms to Multibinit's internal ordering. You need to:

1. **Create atoms** matching the supercell defined in Multibinit
2. **Set masses** corresponding to atom types
3. **Define the pair style** with proper parameters

### Supercell Convention

The supercell size must match the `ncell` parameters in the pair style:

```lammps
pair_style multibinit ncell_x ncell_y ncell_z ngqpt_x ngqpt_y ngqpt_z dipdip flag
```

| Parameter | Description |
|-----------|-------------|
| `ncell_x/y/z` | Supercell dimensions (e.g., 2 2 2) |
| `ngqpt_x/y/z` | Q-point grid for phonon calculation |
| `dipdip` | 0=off, 1=on for dipole-dipole interactions |

### Pair Coefficients

```lammps
pair_coeff * * system_file coefficient_file
```

- `system_file`: Path to `.DDB` file (harmonic part)
- `coefficient_file`: Path to `.xml` file (anharmonic part, optional)

### Example Structure Input

```lammps
# 2x2x2 supercell of BaHfO3 perovskite (40 atoms)
region supercell block 0 2 0 2 0 2
create_box 3 supercell
create_atoms 1 box &
  basis 1 1 &    # Ba at (0,0,0)
  basis 2 2 &    # Hf at (0.5,0.5,0.5)
  basis 3 3 &    # O at (0.5,0.5,0)
  basis 4 3 &    # O at (0,0.5,0.5)
  basis 5 3      # O at (0.5,0,0.5)

# Masses (order must match atom type)
mass 1 137.33  # Ba
mass 2 178.49  # Hf
mass 3 16.00   # O
```

## Dynamics Examples

### NVT Ensemble (Canonical)

Temperature control with Nosé-Hoover thermostat:

```lammps
# NVT at 300K
pair_style multibinit 2 2 2 ngqpt 2 2 2 dipdip 0
pair_coeff * * BaHfO3.DDB BaHfO3.xml

velocity all create 300.0 12345
fix 1 all nvt temp 300 300 0.1

thermo 100
thermo_style custom step temp pe ke etotal press vol

timestep 0.001
run 5000
```

**Output interpretation**:
- Temperature should fluctuate around 300K
- Energy (pe + ke) conserved on average

### NVE Ensemble (Microcanonical)

Energy-conserving dynamics:

```lammps
# NVE dynamics
pair_style multibinit 2 2 2 ngqpt 2 2 2 dipdip 0
pair_coeff * * BaHfO3.DDB

velocity all create 300.0 12345
fix 1 all nve

thermo 50
thermo_style custom step temp pe ke etotal press

timestep 0.001
run 5000
```

**Output interpretation**:
- Total energy (etotal) should be constant
- Temperature will fluctuate
- Good test of force accuracy

### NPT Ensemble (Isothermal-Isobaric)

Pressure control with barostat:

```lammps
# NPT at 300K, 0 pressure
pair_style multibinit 2 2 2 ngqpt 2 2 2 dipdip 0
pair_coeff * * BaHfO3.DDB

velocity all create 300.0 12345
fix 1 all npt temp 300 300 0.1 iso 0.0 0.0 10.0

thermo 100
thermo_style custom step temp pe ke etotal press vol

timestep 0.001
run 5000
```

**Parameters**:
- `temp Tstart Tend Tdamp`: Temperature with damping parameter
- `iso Pstart Pend Pdamp`: Isotropic pressure with damping

**Output interpretation**:
- Pressure should fluctuate around target
- Volume adjusts to equilibrium
- Temperature controlled by thermostat

**Note**: Start with harmonic potential only (no XML) for NPT to ensure stability. Anharmonic terms can cause instability with barostats.

## Post-Processing

### Trajectory Analysis

Dump trajectory for analysis:

```lammps
dump 1 all custom 100 trajectory.lammpstrj id type x y z vx vy vz
dump_modify 1 sort id
```

### Thermodynamic Properties

Log output columns:

| Column | Description | Units |
|--------|-------------|-------|
| `step` | Timestep | - |
| `temp` | Temperature | K |
| `pe` | Potential energy | eV |
| `ke` | Kinetic energy | eV |
| `etotal` | Total energy | eV |
| `press` | Pressure | bars |
| `vol` | Volume | Å³ |

### Analysis with Python

```python
import numpy as np
import matplotlib.pyplot as plt

# Load LAMMPS log
data = np.loadtxt('log.lammps', skiprows=2)
step = data[:, 0]
temp = data[:, 1]
energy = data[:, 4]

plt.figure(figsize=(10, 4))
plt.subplot(121)
plt.plot(step, temp)
plt.xlabel('Step')
plt.ylabel('Temperature (K)')
plt.title('NVT Simulation')

plt.subplot(122)
plt.plot(step, energy)
plt.xlabel('Step')
plt.ylabel('Total Energy (eV)')
plt.title('Energy Conservation')

plt.tight_layout()
plt.savefig('analysis.png')
```

### Radial Distribution Function

```lammps
compute rdf all rdf 100 1 1 1 2 1 3 2 2 2 3 3 3

```

### Mean Square Displacement


## Troubleshooting

### Common Issues

#### "Atom mapping failed"

**Cause**: LAMMPS atom positions don't match Multibinit reference structure.

**Solution**: Ensure lattice parameter and supercell match the DDB file.


#### "Plugin not found"

**Cause**: Library path not set.

**Solution**: 
```bash
export LD_LIBRARY_PATH=/path/to/abinit/lib:$LD_LIBRARY_PATH
```

### Performance Tips

1. **Use MPI**: Parallel runs significantly faster for large systems

## API Reference

### Pair Style Command

```
pair_style multibinit ncell_x ncell_y ncell_z ngqpt_x ngqpt_y ngqpt_z dipdip_flag
```

**Arguments**:
- `ncell_*`: Supercell dimensions (must match structure)
- `ngqpt_*`: Q-point grid for long-range interactions
- `dipdip`: 0 (off) or 1 (on) for dipole-dipole interactions

### Pair Coeff Command

```
pair_coeff * * system_file [coefficient_file]
```

**Arguments**:
- `system_file`: Path to `.DDB` file (required)
- `coefficient_file`: Path to `.xml` file (optional)

### Units

All calculations use LAMMPS "metal" units:
- Distance: Å
- Energy: eV
- Force: eV/Å
- Pressure: bars
- Time: ps
- Mass: g/mol
